# МаркетБот — PMF Pipeline

## Что это
Личный инструмент для прохождения 10 этапов PMF (Product-Market Fit) анализа. Три входа (Telegram личка, Telegram группа, Web UI), одно ядро обработки. Роутинг через LLM: draft → polish.

## Аудитория и доступ
- **Владелец** (owner_id в config.yaml) — полный доступ везде
- **Гости** — приглашённые участники через invite-токены, работают в личке бота в рамках одного назначенного проекта
- **Участники группы** — пассивный сбор контекста из привязанной супергруппы

## Каналы ввода

### Telegram Bot (личка)
- Команды: `/new`, `/use`, `/projects`, `/status`, `/export`, `/continue`, `/skip`
- Artifact-команды: `/hypothesize`, `/brainstorm`, `/rate` — сохраняют результаты в папки проекта
- Голосовые сообщения — транскрибация через faster-whisper
- Детекция интентов — предлагает инлайн-кнопку с нужной командой если текст похож на запрос
- Персистентное меню проекта — ReplyKeyboardMarkup с кнопками текущего проекта
- Сессия проекта сохраняется между перезапусками (`data/user_sessions.json`)
- `/feedback` — сохраняет замечание в `data/bot_feedback.md`
- `/share <проект>` (owner) — генерирует invite-ссылку для гостя (48ч, одноразовая)
- `/revoke <user_id>` (owner) — отзывает гостевой доступ

### Telegram Group Listener
- Пассивный сбор сообщений из привязанных супергрупп в `group_context.md`
- Привязка группы к проекту через `group_links.json`
- Обращения к боту через @ — отвечает в контексте проекта

### Web UI (FastAPI v4.0, порт 8080)
- Запуск PMF-этапов, AI-чат с RAG, PMF Score, Activity Feed, голосовые заметки
- Мониторинг баланса OpenRouter с виджетом в хедере (4 знака, автообновление 1 мин)
- Управление гостями: таблица участников групп, выдача/отзыв доступа, таймлайн активности
- Форма обратной связи встроена в попап AI-ассистента

## Ключевые принципы
- Единый источник истины — файловая система (`projects/{project_name}/`)
- Жёсткий линейный пайплайн: 10 этапов (0_setup → 10_iterate), все через draft → polish
- Artifact-режимы (hypothesize/brainstorm/rate) работают только с project_summary, без истории чата
- Chat-режим использует полный контекст (summary + group_context tail), но не перечисляет старые артефакты
- Per-user asyncio.Semaphore — сериализует обработку всех updates от одного пользователя (включая callbacks)
- Один экземпляр бота — watchdog через PID-файл + heartbeat каждые 30 мин

## Хранение обратной связи
`data/bot_feedback.md` — единый файл, 4 источника:
- 📱 личка — `/feedback` в личном чате
- 👥 группа — `/feedback` в группе
- 🌐 webui — форма в AI-попапе
- 💬 claude-session — записи из сессий разработки
